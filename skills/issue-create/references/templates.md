# Issue 模板

## Feature / 其他类型

```markdown
## 概述

{1-3 句话描述本 Issue 要完成的工作}

## 背景

{为什么需要做。如果有相关 Issue 可引用，例如"Issue #X 已实现 Y，但 Z 仍缺失"}

## 验收标准

- [ ] {验收标准 1}
- [ ] {验收标准 2}
- [ ] ...

## 范围

**包含：**
- {本 Issue 涵盖的范围}

**不包含：**
- {本 Issue 明确不包含的范围}

## 测试计划

- {如何验证每个验收标准}
```

## Bug 类型

```markdown
## 问题描述

{简要描述观察到的问题现象}

## 复现步骤

1. {步骤 1}
2. {步骤 2}
3. {步骤 3}

## 期望行为

{应该发生什么}

## 实际行为

{实际发生了什么}

## 环境信息

- OS / 浏览器 / 版本：{...}
- 相关日志或截图：{如有}

## 验收标准

- [ ] {修复后应满足的条件 1}
- [ ] {修复后应满足的条件 2}
```
